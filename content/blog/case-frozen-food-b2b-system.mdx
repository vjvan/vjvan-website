## 這家供應商原本遇到什麼問題？

這是一家專做 B2B 冷凍食品批發的供應商，客戶主要是餐廳、小吃店、便當店。每天的日常就是接單、備貨、出貨。

但接單這件事，幾乎全靠人工：

- **電話接單**：客戶打電話報品項和數量，接單人員手寫記錄，聽錯、寫錯每週發生 2-3 次
- **LINE 訊息接單**：客戶用 LINE 傳文字「我要鯖魚 5 片、花枝丸 3 包」，員工要自己整理成訂單
- **客戶管理靠記憶**：誰常買什麼、多久沒來、該不該提醒補貨，全靠老闆的記憶和經驗
- **通知靠人工**：訂單確認、出貨通知、到貨提醒，全部手動一個一個發
- **營運數據看不到**：這個月賣了多少、哪個品項最好賣、哪個客戶快流失，要等月底手動統計

老闆每天花大量時間在這些重複性工作上，沒有時間去開發新客戶或優化產品線。

### 前台：LINE LIFF 自助下單

客戶打開 LINE 就能看到完整商品目錄（冷凍魚片、丸類、火鍋料、醬料等），選好品項、規格、數量，一鍵送出訂單。

關鍵設計：
- **不用下載 App**：客戶在 LINE 裡點圖文選單就進入訂貨頁面
- **商品分類清楚**：按品類瀏覽，有圖片和規格說明，不會點錯
- **一鍵重複訂購**：客戶可以從歷史訂單快速重下單，B2B 回購場景最需要這個
- **多單位切換**：同一商品可以按「片」或按「箱」下單，系統自動換算

### 後台：CRM 管理儀表板

老闆打開管理後台就能看到：
- **今日訂單**：即時同步，不用等人整理
- **客戶消費頻率**：誰每週固定補貨、誰已經 14 天沒下單
- **商品排行**：哪些品項賣最好、哪些該補庫存
- **營收趨勢**：月對月比較，不用等月底手算
- **客戶健康度**：紅黃綠燈標示流失風險，一目了然

### 自動化：n8n 流程引擎

把所有重複性通知和追蹤交給系統：
- **訂單確認**：客戶下單後自動推播確認通知
- **出貨提醒**：備貨完成自動通知客戶預計到貨時間
- **補貨提醒**：根據客戶購買週期，自動推播「該補貨了」
- **流失喚回**：超過設定天數未下單的客戶，自動發送喚回訊息

### AI 智慧推薦

系統會根據客戶的歷史訂單，分析購買頻率和偏好，自動推薦可能需要補貨的品項。不是隨機推薦，而是基於加權移動平均、趨勢分析和信心度計算的預測。

## 這套系統的技術架構是什麼？

| 層級 | 技術 | 角色 |
|------|------|------|
| 前台 | LINE LIFF (React) | 客戶自助下單介面 |
| 資料庫 | Supabase (PostgreSQL) | 訂單、客戶、商品資料，含 Row Level Security |
| 後台 | Next.js | CRM 管理儀表板 |
| 自動化 | n8n | 通知推播、補貨提醒、流失喚回 |
| 部署 | Vercel + Cloudflare | 全球 CDN、自動部署 |

整套系統的月維護成本極低：Supabase 免費方案可涵蓋初期用量，Vercel 靜態部署免費，LINE 基本方案 NT$800/月 含 3,000 則推播。

## FAQ

### 這套系統適合多大規模的供應商？

從月出貨 50 筆到 500 筆的供應商都適合。系統架構可以隨業務成長擴充，不需要到一定規模才值得導入。越早導入，越早開始累積客戶數據和購買行為分析。

### 客戶不會用 LINE LIFF 怎麼辦？

實測發現，B2B 客戶的轉換率比預期高很多（首月 83%）。因為他們本來就在 LINE 裡下單，只是從「打字傳訊息」變成「點選商品送出」，操作更簡單而不是更複雜。少數不習慣的客戶，電話接單仍然保留，員工可以在後台代為下單。

### 系統能跟現有的 ERP 或會計系統串接嗎？

可以。透過 n8n 自動化流程，訂單資料可以自動匯出為 ERP 相容格式，或直接透過 API 回寫。目前已驗證與正航 ERP 的資料對接流程。

### 導入需要停業或暫停營運嗎？

不需要。系統上線採並行切換：先讓部分客戶試用 LIFF 下單，原有的電話和 LINE 訊息接單照常運作。等大多數客戶都轉到線上後，再逐步減少人工接單。整個過程不影響出貨。

## 結論

對冷凍食品供應商來說，B2B 補貨系統不是「有了比較好」的東西，而是直接影響每天接單效率和客戶滿意度的營運基礎設施。

這個案例證明了一件事：不需要百萬預算，4 週就能上線一套讓客戶自己下單、系統自動跑、老闆即時看數據的完整營運系統。
